\section{Introducción}
El diseño de interfaces móviles de calidad demanda la adopción de esquemas visuales fluidos y adaptables (responsivos). Esto asegura que los elementos gráficos no sufran desbordamientos de pantalla (\textit{overflow}) o recortes debido a las grandes diferencias en resoluciones, relaciones de aspecto y el despliegue del teclado virtual del sistema operativo. 

En este reporte se documenta el proceso y el código fuente empleado para elaborar una versión responsiva y premium de la pantalla de inicio de sesión de la aplicación móvil de \textbf{Banco Pichincha}. Esta implementación respeta los lineamientos corporativos estéticos de la marca e introduce una experiencia de usuario completamente responsiva sobre la plataforma Android.

\section{Propiedades de la Interfaz y Responsividad}
Para lograr que la interfaz se adapte perfectamente a cualquier dispositivo, se implementaron los siguientes patrones arquitectónicos en la vista XML:
\begin{enumerate}
    \item \textbf{ScrollView como Raíz}: El layout principal está envuelto en un contenedor \texttt{ScrollView} configurado con \texttt{android:fillViewport="true"}. Esto asegura que todo el lienzo sea desplazable de manera automática cuando el teclado virtual aparece o en pantallas con menor altura física.
    \item \textbf{Restricciones Horizontales Fluídas (ConstraintLayout Chain)}: Las tres tarjetas que definen las opciones principales de acceso (\textit{Usuario y contraseña}, \textit{Huella / Face ID} y \textit{Pin de 6 dígitos}) se maquetaron mediante una cadena horizontal (\textit{Horizontal Chain}) usando anchos dinámicos de \texttt{0dp} e inter-espaciado controlado. Esto distribuye el espacio físico disponible proporcionalmente sin importar el ancho del dispositivo (tanto en modo vertical como horizontal).
    \item \textbf{Recursos de Fondo Premium y Ripple Effect}: Se diseñó un archivo XML drawable selector para las tarjetas (\texttt{card\_background.xml}) que proporciona bordes curvos de 16dp y un efecto visual de onda (\textit{ripple}) al ser presionadas.
\end{enumerate}

\subsection{Interactividad (MainActivity.kt)}
El archivo controlador en Kotlin se encarga de dar retroalimentación visual al usuario en pantalla mediante mensajes flotantes temporales \texttt{Toast} cuando interactúa con las opciones.

\begin{lstlisting}[language=Java, caption={Implementación del controlador interactivo: MainActivity.kt}]
package com.example.iniciopichincha

import android.os.Bundle
import android.widget.LinearLayout
import android.widget.TextView
import android.widget.Toast
import androidx.activity.enableEdgeToEdge
import androidx.appcompat.app.AppCompatActivity
import androidx.core.view.ViewCompat
import androidx.core.view.WindowInsetsCompat

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        enableEdgeToEdge()
        setContentView(R.layout.activity_main)
        
        ViewCompat.setOnApplyWindowInsetsListener(findViewById(R.id.main)) { v, insets ->
            val systemBars = insets.getInsets(WindowInsetsCompat.Type.systemBars())
            v.setPadding(systemBars.left, systemBars.top, systemBars.right, systemBars.bottom)
            insets
        }

        // Referencias a los componentes interactivos
        val cardUserPassword = findViewById<LinearLayout>(R.id.cardUserPassword)
        val cardBiometrics = findViewById<LinearLayout>(R.id.cardBiometrics)
        val cardPin = findViewById<LinearLayout>(R.id.cardPin)
        
        val optionLocate = findViewById<LinearLayout>(R.id.optionLocate)
        val optionKey = findViewById<LinearLayout>(R.id.optionKey)
        val optionCall = findViewById<LinearLayout>(R.id.optionCall)
        
        val linkOtherUser = findViewById<TextView>(R.id.linkOtherUser)

        // Asignación de Click Listeners
        cardUserPassword.setOnClickListener {
            showToast("Abriendo inicio de sesión con Usuario y Contraseña")
        }

        cardBiometrics.setOnClickListener {
            showToast("Activando lector de Huella / Face ID")
        }

        cardPin.setOnClickListener {
            showToast("Abriendo inicio de sesión con Pin de 6 dígitos")
        }

        optionLocate.setOnClickListener {
            showToast("Buscando agencias y cajeros cercanos...")
        }

        optionKey.setOnClickListener {
            showToast("Generando Clave Digital...")
        }

        optionCall.setOnClickListener {
            showToast("Llamando a atención al cliente...")
        }

        linkOtherUser.setOnClickListener {
            showToast("Cargando formulario para ingresar con otro usuario")
        }
    }

    private fun showToast(message: String) {
        Toast.makeText(this, message, Toast.LENGTH_SHORT).show()
    }
}
\end{lstlisting}

\section{Conclusión}
El uso combinado de \texttt{ScrollView} como elemento raíz y cadenas horizontales (\textit{Chains}) en \texttt{ConstraintLayout} demuestra ser la solución más robusta y estándar para el diseño de interfaces móviles de registro/inicio de sesión. Se logró emular la experiencia estética de la App de Banco Pichincha con retroalimentación visual óptima y responsividad completa.
